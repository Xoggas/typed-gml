# TypedGML

TypedGML is a statically typed language that compiles to **GML (GameMaker Language)**.

It exists for one simple reason:

> GameMaker is great.
> Writing large projects in raw GML is not.

---

## ✨ What is this?

TypedGML is an attempt to make GameMaker development feel **modern**, **predictable**, and **actually scalable**.

No hacks.
No runtime magic.
No “hope this doesn’t break later”.

Just a language that lets you **build things properly** — and still run them in GameMaker.

---

## 🧠 Philosophy

TypedGML is built around a few core ideas:

### ⚡ Compile-time over runtime

If something can be checked earlier — it should be.

Errors are:

* caught before the game runs
* visible immediately
* not hidden inside some random Step event at runtime

---

### 🧩 Structure matters

Big projects need structure.

TypedGML pushes you towards:

* clear boundaries
* predictable code
* readable systems

Not because it forces you — but because it **makes the right way the easy way**.

---

### 🚫 No magic

No hidden systems.
No runtime frameworks.
No “engine inside the engine”.

What you write is what gets generated.

If something happens — you can trace it.

---

### 🎯 GameMaker-first

This is not a general-purpose language.

TypedGML is designed specifically for:

* GameMaker
* GML runtime
* real production constraints

Everything exists for a reason.
If it doesn’t fit GameMaker — it doesn’t belong here.

---

### 🧘 Simplicity over cleverness

No unnecessary abstractions.
No over-engineering.
No “look how smart this is”.

Just:

* clear rules
* consistent behavior
* predictable output

---

## 🚀 What it gives you

### 🛡️ Safety

* Strong typing
* Compile-time validation
* Fewer runtime surprises

---

### 🧼 Clean code

* Less chaos
* More clarity
* Easier refactoring

---

### 🧠 Better thinking

You stop writing:

> “code that works somehow”

And start writing:

> “systems that make sense”

---

### 🔄 Control

You’re still in control of:

* what gets generated
* how it behaves
* how it maps to GML

No black boxes.

---

## ⚠️ What this is NOT

Let’s be very clear:

* ❌ Not a replacement for GameMaker
* ❌ Not a runtime framework
* ❌ Not Unity inside GML
* ❌ Not trying to hide how GML works

TypedGML is just a **tool** — a very opinionated one.

---

## 🔥 Why it exists

Because at some point you hit this:

* the project grows
* the code becomes fragile
* changes become scary
* debugging becomes pain

And you realize:

> “I don’t need more features.
> I need better foundations.”
